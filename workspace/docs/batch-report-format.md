# Batch Report Format

**Location:** `workspace/reports/BATCH-REPORT-<YYYYMMDD-HHMMSS>.md`

Batch reports are generated by the reviewer after each launcher run.
They summarise the state of the task queues and flag items needing manager attention.

---

## Header Block

```
# Batch Review Report

Created: MM/DD/YYYY HH:MM:SS

Todo: <count>
In Progress: <count>
Review: <count>
Done: <count>
Failed: <count>
Locks Remaining: <count>
Run Logs: <count>
```

### Field definitions

| Field | Description |
|---|---|
| `Todo` | Tasks still in `tasks/todo/` |
| `In Progress` | Tasks currently in `tasks/in_progress/` |
| `Review` | Tasks awaiting manager review in `tasks/review/` |
| `Done` | Tasks in `tasks/done/` |
| `Failed` | Tasks in `tasks/failed/` |
| `Locks Remaining` | Lock files still present in `locks/` — non-zero is a warning |
| `Run Logs` | Number of run log files in `runs/` created during this batch |

---

## Per-Task Summary (recommended)

Each task that moved state during the batch should appear in a table:

```
## Task Transitions

| Task ID | Worker | Display Name | From | To |
|---|---|---|---|---|
| SAMPLE-002 | heavy_worker | Forge | todo | review |
| SAMPLE-001 | debug_worker | Scout | todo | review |
```

---

## Manager Review Notes

Free-form section. Common notes:

- Confirm locks were released (`Locks Remaining` should be 0 after a clean run).
- Review any tasks with `Ending status: failed`.
- Move tasks from `review` to `done` after human sign-off.
